# IT Sicherheit#
## CIA Triad
The CIA Triad is the 3 pillar of IT Sicherheit
- Confidentiality: People without the right permission are not allowed to see the data
- Integrity: Data remains accurate and intact
- Availability: Data should have a high uptime. (DDoS attack

| Name  | What it is about | What kind of attacks it tries to prevent | How it prevents error |
| --- | --- | --- | --- |
| **Confidentiality** | Ensuring that information is only accessible to authorized people | Unauthorized access, eavesdropping, data leaks, spying (e.g. sniffing, man-in-the-middle) | Encryption of data |
| **Integrity**       | Ensuring that data is correct, complete, and not altered  | Data manipulation, tampering, unauthorized modification (e.g. injection attacks, file corruption) |Kryptographischer Hash<br>Massage Authentication Codes (MAC)<br>>Tamper-Resistant Security Module (TRSM) |
| **Availability**    | Ensing systems and data are accessible when needed | Service disruption, system overload, downtime (e.g. DDoS attacks, hardware failures) |DDOS Protection<br>Automatic Scaling|

## Further Schutzziel (PANZ)
- Privatsphäre: Teile oder sämtliche Kommunikationswege eines Nutzers geheim halten
- Authenticität: Identitätsnachweis des Kommunikationspartners, sowie Nachweis, dass die Daten von genau diesem kommen
- Nichtabstreitbarkeit: Die Kommunikation (Informationsaustausch) kann von einer beteiligten Instanz gegenüber Dritten im Nachhinein nicht abgestritten werden
- Zurechenbarkeit: Eine durchgeführte Handlung kann einem Kommunikationspartner eindeutig zugeordnet werden

## Privacy
- **Psuedoanonimity**: Personal data is replaced with identifiers (e.g. IDs, usernames). The data can still be traced back to a person if additional information (a key or mapping) is available.
- **Anonomity**: Data is processed so that individuals cannot be identified at all, even indirectly. This is often achieved through aggregation or strong data transformation.

# Kryptographie
Kryptographie is the process of encrypting data, such that only a user with the right *key* can unscramble the data.

**Difference with hashing**

While both hashing and encryption jumbles the data, the difference is that hashing cannot be unscrambled, and the data is lost.  
Encryption is required if the data needs to be pieced back together again.

In this topic we will be using the following notation

$$
\text{Klartext (Cleartext)} = x\\
\text{Geheimtext (Ciphertext)} = y\\
\text{Schlüssel (Key)} = k \in \text{Schlüsselmenge (Keyspace)} = K\\
\text{Verschlüsselung (Encryption)} = e()\\
\text{Entschlüsselung (Decryption)} = d()
$$

## Symmetric Kyrptographie
Symmetric Kryptographie is a encryption process where the sender and the reciever both have the same key.

- Encryption of a decrypted text is cipertext
- Decryption of a encrypted text is cleartext
$$
e_k(d_k(y)) = y\\
d_k(e_k(x)) = x\\
$$

- Therefore Encryption is the reverse of Decryption
$$
e_k() \xrightarrow{f'}d_k()
$$

### Classic Example
#### Caeser Chiffre
The Caeser Cipher is the shifting of the mapping of all alphabet by a fix amount. The mathematical notation of the caeser cipher is 
$$
𝑒_𝑘(𝑥) = \text{𝑥 + 𝑘 𝑚𝑜𝑑 26}\\
𝑑_𝑘(𝑦) = \text{𝑦 − 𝑘 𝑚𝑜𝑑 26}
$$

#### Substitution Chiffre
The substituion Cipher is the mapping of an alphabet with another **random** alphabet, but each alphabet can only be used **once**

Step by step:
1. For A, you can choose any of 26 letters
2. For B, only 25 letters left
3. For C, only 24 letters left
...
26. For Z, you only have 1 letter left

To build the table of mapping, there are a total of 26! different combination.

**Cons**
- Language has patterns → frequency analysis breaks it easily

#### Stream Cipher
Encrypt data bit by bit or character by character like a continuous stream

You take your plaintext and combine it with a key stream (random-looking sequence)

Characteristic:
- Fast, low latency
- Good for real-time communication (e.g. voice, video)
- No fixed block size
#### Block Cipher
Encrypt data in fixed-size blocks (e.g. 128 bits)

You take a chunk of data and encrypt the whole block at once


Characteristic:
- Works in chunks (blocks)
- More structured
- Used in many standard algorithms (like AES)

# Random Number Generators:
| Name of RNG | What it does | Pros | Cons |
|---|---|---|---|
| TRNG (True Random Number Generator) | Generates randomness from physical processes (e.g. noise, radiation) | Truly random, unpredictable, high security | Slow, requires special hardware, not always available |
| PRNG (Pseudo Random Number Generator) | Generates numbers using deterministic algorithms based on a seed | Fast, efficient, reproducible | Not truly random, predictable if seed is known |
| CSPRNG (Cryptographically Secure PRNG) | A PRNG designed to be secure and unpredictable even if partially known | Secure for cryptography, hard to predict, practical speed | Slower than PRNG, more complex |

# One-Time Pad (OTP)

## What it is
A stream cipher where:
- Plaintext is combined with a key (same length)
- Typically using XOR (⊕)

## How it works
**XOR Table**

|$\oplus$|True|False|
|---|---|---|
|True|0|1|
|False|1|0|

1. Notice that XOR encryption is a reversable operation
2. Rule:
    - If key bit = 0 → keep the bit
    - If key bit = 1 → flip the bit
```js
1 ⊕ 1 = 0
0 ⊕ 1 = 1

// So
K ⊕ K = 0
0 ⊕ P = P
```
Let:
- P = plaintext
- K = key
- C = ciphertext

Formulat
- Encryption: C = P ⊕ K
- Decryption: C ⊕ K = (P ⊕ K) ⊕ K = P

Expanding on Decryption:
1. (P ⊕ K) ⊕ K
2. = P ⊕ (K ⊕ K)
3. = P ⊕ 0
4. = P



Plaintext ⊕ Key = Ciphertext  
Ciphertext ⊕ Key = Plaintext

## Unconditional Security
- Cannot be broken, even with infinite computing power
- Every possible plaintext is equally likely

---

## Requirements (must ALL hold)

1. **Key is truly random**
   - Generated by a real random source (TRNG)
   - Not pseudo-random

2. **Key is secret**
   - Only known to sender and receiver

3. **Key is used only once**
   - Never reused

---

## Why it’s not practical
- Key must be as long as the message
- Key distribution is difficult
- Key management is complex

---

## Recap
One-Time Pad = perfectly secure encryption, if the key is random, secret, and never reused.
